# Development Situation List

This document contains the complete list of development situations extracted from the workflow state diagram.

## Situations

1. **IntentDefined**
   - The development intent has been clearly defined

2. **IntentDefinedFail**
   - The intent is unclear and needs clarification

3. **ProblemSelected**
   - A specific problem has been selected to work on

4. **AcceptanceDefined**
   - Acceptance criteria have been defined for the problem

5. **FeasibilityChecked**
   - Feasibility of the solution has been checked

6. **DesignReady**
   - Design with state transitions has been completed

7. **TaskBreakdown**
   - Tasks have been broken down into executable units

8. **Implementing**
   - Currently implementing a task

9. **Verifying**
   - Verifying that the implementation meets acceptance criteria

10. **Verified**
    - Implementation has been verified to meet acceptance criteria

11. **Released**
    - The work has been released/delivered

12. **FeedbackCollected**
    - Feedback has been collected after release

13. **Learned**
    - Learning from feedback has been completed

## Terminal States

- **[Start]** - Initial state (not a situation, but entry point)
- **[End]** - Final state (not a situation, but exit point)

## Special State

- **Undetermined** - Current situation cannot be determined or is ambiguous
